Why SVG?

*Can't you do everything with HTML + CSS?*

HTML5 is awesome, you can do so much with it! Canvas is another interesting option for rendering documents, but is much more complex and less portable

Why SVG?

  • HTML + CSS alone does not provide everything needed to render documents
  • Converted assets are be standard, portable, and lightweight
  • Rendering fidelity is incredibly high

Inline SVG FTW!

Inline SVG is part of the [HTML 5 spec](http://www.w3.org/html/wg/drafts/html/master/single-page.html#svg)!

DOMParser !== inline SVG

Inline SVG is great for static content rendered into the HTML page, but that work work here In order to dynamically embed SVG content, it's necessary to use DOMParser or innerHTML to insert SVG content At least in Chrome, invoking window.DOMParser bypasses the threaded html parser in blink, which causes a synchronous parse of the SVG content In fact, the threaded html parser is not exposed to JS at all!

OK, inline SVG won't work. Now what?

There are several ways to dynamically embed SVG content in HTML:

  • inline <svg> using DOMParser
  • <img>
  • <iframe>, <object>, or <embed>
